Architecture Overview

Frontend on GitHub Pages + Backend on Railway

Frontend (GitHub Pages)

  • • Static HTML/CSS/JavaScript
  • • Tailwind CSS for styling
  • • AJAX calls to Railway backend
  • • Free hosting on GitHub Pages
  • • Custom domain support

Backend (Railway)

  • • PHP API endpoints
  • • MySQL database
  • • PDO for database operations
  • • CORS enabled for frontend
  • • Free tier available

Setup Instructions

How to deploy this architecture

1. Deploy Backend to Railway

# Create railway.json
{
"build": {
"builder": "NIXPACKS"
},
"deploy": {
"startCommand": "php -S 0.0.0.0:$PORT"
}
}

2. Deploy Frontend to GitHub Pages

1. Push this folder to GitHub repository
2. Go to Settings → Pages
3. Select source branch (main)
4. Update API_BASE_URL in JavaScript

3. Environment Variables

Railway Backend:
• DB_HOST, DB_NAME, DB_USER, DB_PASS
• CORS_ORIGIN (your GitHub Pages URL)
